### What was done
Debugged and fixed the complete photo-to-finding attachment pipeline. Root causes found and fixed:

1. **System prompt gap (migration 0030)**: Old Technician prompt never mentioned `add_finding`, `set_report_summary`, or `[Photo N: url]` markers. New prompt has explicit PHOTO HANDLING section.

2. **Private S3 bucket + SigV2 = double failure**:
   - `chat.py` was sending raw private S3 URLs in `image_urls` â†’ Anthropic returned 400 "Unable to download the file" when photos were attached
   - `reports.py` was returning raw private S3 URLs in findings `photo_url` â†’ iOS `AsyncImage` got 403 Forbidden
   - Fix: added `_presign_s3_urls()` helper in `chat.py` to generate 1-hour presigned URLs before Anthropic API call; made `_to_staff_detail` async and generate 7-day presigned URLs at serve time

3. **Wrong S3 signature version (SigV2)**: `generate_presigned_url` was using legacy Signature V2 (`AWSAccessKeyId` format) which is rejected with 403 for buckets created after 2020. Fixed by adding `Config(signature_version="s3v4")` to S3 client kwargs.

4. **Presigned URL expiry too long**: 30-day expiry (2,592,000s) > SigV4 max (604,800s = 7 days) â†’ S3 returned 400 `AuthorizationQueryParametersError`. Fixed to 7-day cap.
