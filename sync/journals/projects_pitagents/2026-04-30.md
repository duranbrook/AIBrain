## Session 16:00 — Fix PTT voice navigation bug

### What was done
- Identified root cause of voice control not navigating despite showing connected (green ring):
  - In PTT mode, browser fires `mousedown → mouseup → click` in sequence
  - `mousedown` → `startCapture()` (enables audio track)
  - `mouseup` → `stopCapture()` (commits audio buffer, sends to OpenAI)
  - `click` → `disconnect()` ← **kills session before OpenAI responds with tool call**
  - This is why there was no network activity or console output — the tool call never arrived
- Fixed in `VoiceControlWidget.tsx`: added `mouseDownAt` ref to track hold duration
  - Holds > 200ms (PTT): click event suppresses disconnect
  - Short taps (< 200ms): disconnect works as expected (intentional tap-to-disconnect)
- Deployed to `pitagents.vercel.app`

### Current state
- PTT mode: hold mic to speak → releases send audio to OpenAI → tool call fires → navigation occurs
- Short tap still disconnects
- Debug `console.log` statements still in voice files (intentional — will remove once user confirms navigation works)

## Session resumed — Fix voice multi-step chain
### What was done
- Diagnosed chain-stopping bug: after tool 1 executed (via `#handleOutputItemDone`), the `#handleResponseDone` task computed `executedCount=0` because the `responseId` from `response.output_item.done` might be null, so `#responseToolCounts` was never populated. The chain condition `(pendingCalls.length > 0 || executedCount > 0)` was false → `#requestPostToolResponse` never fired.
- Fix 1: added `|| this.#toolExecutedDuringResponse` to chain condition — this flag is set by `#executeToolCall` and reset on each `response.created`, so it's a reliable proxy for "a tool ran in this response" regardless of responseId.
- Fix 2: changed `tool_choice` from `"required"` to `"auto"` — with required, the model was forced to call tools even when the chain was complete, burning through the depth=8 limit.
- Fix 3: updated INSTRUCTIONS to say "call ALL needed tools in a single response" — model should batch all steps in response 1, then stop on the post-tool response.create.
- Added debug logging: `outputItemDone` (shows responseId from output_item.done), `chain-check` (shows executedCount, pendingCallsLen, toolExecutedDuringResponse), `response.done` (shows responseId from both locations to confirm mismatch hypothesis).
- All committed and deployed to pitagents.vercel.app.
