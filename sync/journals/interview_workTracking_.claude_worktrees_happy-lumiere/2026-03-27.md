### Design Decisions Made

**Project Vision:** Employee work tracking app that quantitatively measures individual performance across three categories:
1. **Code/PR metrics** — PRs pushed/merged, features delivered, complexity
2. **Meeting involvement** — attendance, participation quality (passive/active/leader), conflict resolution
3. **Investigation work** — problem difficulty, time spent investigating issues

**Architecture Decisions:**
| Decision | Choice | Reasoning |
|----------|--------|-----------|
| Tech stack | **Go or Rust compiled daemon + web dashboard** (Option B) | Lightweight, single binary, hard to tamper with as system service |
| Meeting data collection | **Calendar integration + audio transcription + LLM analysis** (Option B = A+D) | Agent on laptop hooks into Zoom/Teams/Meet APIs, transcribes, LLM scores participation quality. Keeps everything on-device. |
| Target platform | **macOS only** (MVP) | Simplest to build; architecture should support cross-platform later via adapter modules |
| Code metrics collection | **GitHub/GitLab API polling** (Option A) | Daemon periodically queries platform API for PR activity, diff stats, review comments, merge history. Simpler, less invasive. |
| Audience model | **Employee + Manager** (Option B) | Employees see own data (self-improvement), managers see team-wide dashboards with comparisons. Reduces surveillance perception. |

**Key App Characteristics:**
- Runs as mandatory background daemon on employee laptops (macOS launchd)
- Cannot be stopped by the employee
- Pushes daily journal to a dedicated git repository
- Designed for long-term evaluation (6-12 months)
- Measurement criteria are expected to evolve over time
