### Architecture delivered
```
AnthropicClient (singleton @Bean in ClaudeConfig)
       ├── ClaudeService          → messages API (ask)
       └── ClaudeFileService      → Files API + Document persistence
               ├── S3Client       → fetch file bytes by s3_key
               └── DocumentRepository → persist anthropic_file_id
```

Endpoints:
- `POST /api/ai/ask` — plain text prompt
- `POST /api/ai/files/register/{documentId}` — register from S3 (idempotent)
- `POST /api/ai/files/register/{documentId}/bytes` — register from raw upload (idempotent)

### Key decisions
- `contentType` removed from `registerFromBytes` — Anthropic SDK v2.17.0 `FileUploadParams` doesn't support content-type metadata (verified via javap)
- Field ordering in `Document.kt`: `anthropicFileId` placed between `processingStatus` and `isDeleted` (functionally equivalent, harmless deviation from plan)
- Unit tests only (MockK) — no integration tests since feature not yet wired to any domain flow
