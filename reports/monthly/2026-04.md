## Top 3 Monthly Learnings

1. **Subagent-Driven Development (SDD) is now the proven default for shipping a multi-task sub-project — but the post-implementation code-review/smoke step is non-optional.** April delivered four consecutive sub-projects via SDD without any of them needing to re-justify the approach: pitagents Sub-project 1 Web Chat (15 tasks, 76 backend tests, 2026-04-23); Sub-project 2 Quote Agent (10 tasks, 2026-04-24); Sub-project 3 Mobile (iOS 10 tasks + Android 12 tasks, 21 combined commits, 2026-04-25); and on 2026-04-30 a fourth in the same shape — Voice Layer (Sub-project A) + Report Preview (Sub-project B) shipped end-to-end on web/backend in a single day with full Vercel + Railway deploys. The pattern is mature. *But every single SDD ship this month had bugs that green unit tests didn't catch.* Quote Agent had 4 runtime bugs (4-24); Mobile had backend enum drift undetected by 95 unit tests until iOS/Android consumed the API (4-25); pitagents code review caught empty `intent` field + missing 403 test (4-24); 8 code-quality fixes were caught in pre-merge review on 4-30. **Operational rule: SDD's definition-of-done = subagents land + code review pass + smoke from a non-test consumer**, in that order. Cutting either of the last two steps re-introduces the bugs SDD's per-task tests can't see. Cross-references: daily reports 2026-04-23, 2026-04-24, 2026-04-25, 2026-04-30; W17 weekly rollup `reports/weekly/2026-W17.md`.

2. **"Shipped ≠ smoke-tested" was a five-instance monthly pattern, not a one-off — and the cheapest drift detector is the first consumer in a different runtime.** Five distinct instances in April: (a) Quote Agent's 4 runtime bugs (`3f143cc` arg-order, `be29322` qty field + refetch, `191fbd0` swallowed stream errors, `c11c535` quoteId reload) — unit tests green throughout (4-24); (b) Mobile/backend enum drift — backend used `direction="out"`/`channel="wa"`, mobile expected `"outbound"`/`"sms"`; 95 backend unit tests passed, only the cross-runtime call surfaced it (4-25); (c) Pixel 10a Gherkin smoke caught backend `POST /vehicles/{id}/messages` returning HTTP 500 and `InspectScreen` spec divergence — unit tests had been green for days (4-27, 4-28); (d) iOS Simulator pixel-tap calibration worked one day, broke the next, fragility only visible via repeated runs (4-26 → 4-27); (e) voice multi-step chain bug only surfaced when running the live browser session against the deployed backend (4-30). The 2026-04-28 daily report explicitly flagged "third instance in two weeks"; the month closed at five. Cheapest drift detector across all five: **the first non-test consumer in a different language/runtime** — Gherkin scenario, mobile client, browser session — *not* another batch of same-language unit tests. Promoted to monthly DoD: a feature is "done" when a different-runtime consumer has executed it once, not when unit tests pass. Cross-references: daily reports 2026-04-24, 2026-04-25, 2026-04-27, 2026-04-28, 2026-04-30.

3. **Silent failures cluster at language/protocol/deployment boundaries — make the boundary explicit, because type checkers and unit tests don't see them.** April produced an unusually concentrated set of boundary-silent failures, all of which class together: (a) JWT URL-safe base64 decode silently empty across Swift `Data(base64Encoded:)`, Java `Base64`, Python `b64decode`, JS `atob()` — `email`/`role`/`shop_id` blank for some users, captured cross-platform in `reference_jwt_url_safe_base64.md` (4-26); (b) backend↔mobile enum drift (`out`/`in` vs `outbound`/`inbound`, `wa`/`email` vs `sms`/`email`) — type checkers can't bridge JSON wire format (4-25); (c) presigned S3 URLs used as DB identifiers — every comparison after re-presign silently returns false; capability ≠ identifier, captured in `feedback_presigned_url_pitfalls.md` (4-27); (d) iOS `MessageResponse` Codable silently dropped `external_id`/`sent_at` because they weren't in `CodingKeys` — backend kept sending them, iOS quietly ignored (4-28); (e) Vercel project-level `framework: null` silently downgraded a Next.js deploy to `@vercel/static-build`, deploying only public/ static assets — bundle looked healthy locally; project-name and framework lock-in are server-side state invisible from the repo (4-30); (f) PTT button: browsers always synthesize `click` after `mouseup`, so a `click → disconnect()` handler killed the WebRTC session before OpenAI's tool call returned — no error, no log, just nothing happens (4-30); (g) voice multi-step chain stalled because `responseId` from a streaming event could be null, so the per-response counter map was never populated — chain condition `executedCount > 0` returned false silently (4-30); (h) `passlib 1.7.x` × `bcrypt 4.x` silent runtime hash failure — appeared in both stlab (4-22) and pitagents (4-22/23). Pattern across all eight: **at every language/protocol/deployment/runtime boundary, the most reliable failure mode is silent**. Counter-pattern: make the boundary explicit (smoke tests, schema-contract assertions, redundant decode-and-verify, deploy framework pinning), because the language tooling on either side won't surface the mismatch. Cross-references: daily reports 2026-04-22, 2026-04-23, 2026-04-25, 2026-04-26, 2026-04-27, 2026-04-28, 2026-04-30.
