# Daily Report: 2026-04-08

## Key Learnings (Top 3)

1. **Next.js 16 renames `middleware.ts` → `proxy.ts` (and its export)** — The file must be `src/proxy.ts` exporting `export function proxy`. The old name still works but causes a deprecation warning and appears broken in build output. This is not in training data and caused real build failures. Also: `npx tsc` is broken in Next.js 16 — use `node node_modules/typescript/bin/tsc --noEmit` instead.

2. **Prisma 7 removes `url` from `schema.prisma`** — Database URL now lives in `prisma.config.ts` under `datasource.url`. The constructor requires an explicit adapter: `new PrismaClient({ adapter: new PrismaPg({ connectionString }) })`. No `index.ts` is auto-generated — must create `src/generated/prisma/index.ts` manually with re-exports. These are silent runtime failures if you miss them.

3. **SwiftUI `navigationDestination(item:)` requires `Hashable` conformance** — Any enum used with `navigationDestination(item:)` must conform to both `Identifiable` AND `Hashable`. The compiler error doesn't directly name the missing conformance, making it confusing to diagnose. Source: `LoginView.LoginDestination` error in buildco-ios.
