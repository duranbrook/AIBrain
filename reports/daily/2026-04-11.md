## Key Learnings (Top 3)

1. **Chain-first enrichment strategy scales better than alphabetical/random for large static datasets** — When enriching 3,600+ recycler records with contact data (phone/email/website), starting with major chains (LKQ Keystone, Fenix, B&R Auto Wrecking) is more efficient than working alphabetically. A handful of chains represent a large fraction of total records, so chain-level web searches yield many rows per query. Remaining independents can then be batched by state for parallel agent calls.

This "ship the structure, fill the data later" approach lets the page go live and be useful as a visual directory before enrichment completes. The structure (slug, name, location) already has value without contact info.
