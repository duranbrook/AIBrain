# Daily Report: 2026-04-12 (Sunday)

## Key Learnings (Top 3)

1. **Clerk v7 renamed ClerkProvider redirect props** — `afterSignInUrl`/`afterSignUpUrl` are not valid props in `@clerk/nextjs@7.0.12`. The correct props are `signInForceRedirectUrl` and `signUpForceRedirectUrl`. This caused a silent Vercel build failure (no local warning, only caught in Vercel inspect logs). Always check the Clerk v7 changelog when upgrading from v5/v6. Source: `sync/journals/projects_pitcrews/2026-04-12.md` session 00:00.

2. **BFS adjacency crawling scales YouTube discovery from hundreds to thousands in a single session** — Running 76 BFS waves (8 parallel scripts each) expanded the PitCrew youtuber-outreach list from ~500 to exactly 2,000 channels. The pattern: seed from known channel hubs → crawl /about + /channels pages → batch-collect adjacent recommendations → add and repeat. At ~40–60 net-new channels per productive wave, 2,000 targets required ~30 high-yield waves out of 76 total. This is a reproducible content-discovery strategy for any niche creator outreach list. Source: `sync/journals/projects_pitcrews/2026-04-12.md` sessions 14:00–22:30.

3. **Prisma + vitest: mock the service module, not `@prisma/client`** — `vi.mock('@prisma/client', factory)` does NOT intercept PrismaClient when the package under test has `@prisma/client` as a direct dependency (as opposed to devDep). Prisma's `index.js` is CJS; the vitest ES-module interceptor misses the nested `require()` chain. The reliable fix: mock the service file directly (`vi.mock('../myService', factory)`). Packages where Prisma is a devDependency are unaffected. Source: `sync/memory/-Users-joehe-workspace-projects-pitcrews/feedback_prisma_vitest_mock.md`.
