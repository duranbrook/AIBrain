# Daily Report: 2026-04-23 (Thursday)

## Key Learnings (Top 3)

1. **`passlib 1.7.x` is incompatible with `bcrypt 4.x+` — pin `bcrypt<4.0.0` in every Python project using passlib** — This appeared independently in both `interview_stlab` (2026-04-22) and `projects_pitagents` (2026-04-22/23). It's a silent runtime failure, not a startup error. One-line fix in pyproject.toml: `bcrypt<4.0.0`. *Treat as a standard boilerplate pin for any Python project using passlib going forward.* Source: `sync/journals/interview_stlab/2026-04-22.md`, `sync/journals/projects_pitagents/2026-04-22.md`.

2. **React SSE handlers capture stale state — use functional updaters to fix the stale closure bug** — When accumulating streamed SSE chunks into a message in React, the event handler installed by `useEffect` captures a frozen reference to the state value at mount time. Using a functional updater (`setMsg(prev => prev + chunk)`) instead of `setMsg(msg + chunk)` eliminates the stale closure. This is a classic React footgun that only manifests under streaming conditions. Source: `sync/journals/projects_pitagents/2026-04-23.md` (ChatPanel, Task 13).

3. **Plain SQLAlchemy `chat_messages` table is preferable to LangGraph `AsyncPostgresSaver` for simple chat persistence** — `AsyncPostgresSaver` requires psycopg v3 (breaking API changes vs. v2). For projects already using SQLAlchemy + psycopg v2, adding a plain table is simpler and avoids a major transitive dependency upgrade with no functional benefit for basic chat history. Source: `sync/journals/projects_pitagents/2026-04-23.md` (Key decisions).
