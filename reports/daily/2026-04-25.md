# Daily Report: 2026-04-25 (Saturday)

## Key Learnings (Top 3)

1. **OpenRouter as drop-in OpenAI embeddings provider** — When the real OpenAI key hit a 429 quota during the morning enterprise-AI seed step, setting `OPENAI_BASE_URL=https://openrouter.ai/api/v1` let the OpenAI Python embeddings client route through OpenRouter unchanged. No SDK refactor required. Saved as cross-project memory `reference_openrouter_embeddings.md`. — Cheap, reusable mitigation any time we need to substitute providers.

2. **Subagent-Driven Development scaled to a third sub-project end-to-end in one session** — pitagents Sub-project 3 (Mobile Chat) shipped both iOS (10 tasks, XcodeGen-based) and Android (12 tasks, Compose) in a single session, 21 combined commits. SDD has now driven 3 consecutive sub-projects from plan to merged code (Web Chat → Quote Agent → Mobile). — The pattern's reach is proven; future sub-projects can default to SDD without re-justifying the approach.

3. **Cross-stack work is the cheapest way to catch backend enum drift** — Mobile clients revealed that backend uses direction `"out"/"in"` (not `"outbound"/"inbound"`) and channel `"wa"/"email"` (not `"sms"/"email"`). Unit tests were green; only a second-stack consumer surfaced these undocumented values. — Worth budgeting cross-stack smoke calls early in any multi-client API project.
