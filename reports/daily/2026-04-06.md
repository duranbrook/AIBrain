# Daily Report: 2026-04-06

## Key Learnings (Top 3)

1. **Next.js 16 + Prisma 7 both have significant breaking changes** — Next.js 16 renamed `middleware.ts` → `proxy.ts` (and `export function middleware` → `export function proxy`). Prisma 7 removed `url` from schema; DB config moves to `prisma.config.ts` with `@prisma/adapter-pg` in the client constructor. These are silent failures — builds pass until runtime. Reference: `sync/journals/business/2026-04-06.md`

2. **Dual-mode auth pattern (cookies + Bearer JWT) cleanly supports web and mobile from a single backend** — `getSession()` accepts an optional `bearerToken` param; if present, it bypasses cookie lookup. Mobile uses `expo-secure-store` + Bearer headers; web uses httpOnly cookies. No separate auth service needed. Reference: `sync/journals/business/2026-04-06.md`

3. **Admin bootstrap pattern: allow first promote call when `adminCount === 0`** — avoids the chicken-and-egg problem (can't promote anyone to admin if there are no admins to grant the permission). No manual DB seeding required. Reference: `sync/journals/business/2026-04-06.md`
