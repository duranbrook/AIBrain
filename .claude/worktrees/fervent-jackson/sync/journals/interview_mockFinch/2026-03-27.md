## Session 04:00 — Temporal Workflow Wiring & Client Portal Actions

### What was done

**Temporal Infrastructure Fixes**
- Fixed Temporal server crash — removed `DYNAMIC_CONFIG_FILE_PATH` from docker-compose.yml (file didn't exist in the container image)
- Added `temporal.enabled=true` to application.yml so workflow endpoints load
- Registered Jackson Kotlin module on Temporal's data converter — fixed `InvalidDefinitionException` when deserializing Kotlin data classes

**Workflow Wiring (CaseService → Temporal)**
- Created `CaseWorkflowService` — wraps all workflow operations (start, signal, query) with graceful fallback when Temporal is disabled
- Auto-start case lifecycle workflow on case creation
- Case status updates automatically signal the running workflow (`phase_advance`)
- Document upload triggers document processing workflow
- `closeCase` activity now persists settlement to DB (was a stub that only logged)

**Workflow Bug Fixes**
- Early termination: every `Workflow.await` now checks `isTerminated()` — close/reject at any phase exits cleanly (was blocking forever)
- Phase stepper UI: terminal states (closed/rejected/litigation) show completed steps green, stopped-at step in slate, skipped steps grey (was all grey)
- Terminal state banner: "Case Closed — stopped at Case Setup" displayed above stepper
- Workflow actions hidden for terminated cases (was showing and throwing errors)
- Signal endpoint updates DB immediately for responsive UI (was waiting for Temporal worker to process)
- `lastActivePhase` tracked in workflow and returned via query for accurate UI rendering after completion

**Stalled-Case Detection**
- Every phase now uses `awaitWithStalledAlert()` — sends in-app notification + email every 2 days if no progress
- Treatment phase keeps its 14-day check-in loop (clinical cadence)
- Zero compute cost while waiting (Temporal durable timers)

**Client Portal Workflow Actions**
- `POST /v1/portal/cases/{caseId}/report-mmi` — client reports MMI, signals workflow, advances case to demand_prep
- `POST /v1/portal/cases/{caseId}/confirm-settlement` — client confirms settlement amount, signals workflow, case settles
- Portal UI: amber MMI card in treatment phase, green settlement card in negotiation phase
- Both endpoints validate phase, handle missing workflow gracefully, update DB directly
- Added `documents:write` permission to client role — clients can upload documents from portal

**Frontend Fixes**
- Null-safe date formatting on CaseDetailPage (5 unguarded `format()` calls on nullable dates)
